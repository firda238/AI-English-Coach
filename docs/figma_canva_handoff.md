## 页面设计目标

本项目页面优化围绕三个目标：

- 让页面好看：使用统一的深色科技风视觉系统，减少默认 Streamlit 的粗糙感。
- 让页面好用：让用户按“选择场景 -> 输入 -> 查看反馈 -> 生成总结”的顺序自然完成练习。
- 让页面清楚：把练习、历史、统计、报告和设置拆成清晰的信息区域，适合比赛项目说明截图。

## 信息架构

页面采用左侧栏 + 顶部状态区 + 主内容标签页结构。

左侧栏负责长期上下文：

- 项目名称
- 场景选择
- 难度选择
- 当前运行模式
- 练习目标
- 常见问题和推荐表达
- 最近保存记录

顶部 Hero 负责当前状态总览：

- 产品名称
- 当前模式
- 当前场景
- 当前难度
- 当前练习进度

主内容分为五个标签页：

- 练习中心：当前任务、输入、音频上传、对话历史、AI 回复、纠错反馈、五维评分。
- 历史记录：筛选、查看、下载和删除练习 JSON。
- 学习统计：练习次数、平均分、最高分、趋势图、场景分布、高频问题。
- 报告导出：Markdown 和 HTML 报告导出。
- 设置说明：API dry-run、live 测试、语音转写测试和自动化测试命令。

## 核心组件

### Hero 状态区

用途：让用户打开页面后立即知道当前练习处于什么模式、什么场景、什么难度、什么进度。

设计要点：

- 使用深色渐变背景。
- 左侧显示产品名称和说明。
- 右侧使用四个状态卡片。
- 状态卡片文字必须短，不放长段落。

### 练习任务卡

用途：把“现在该练什么”放在输入区之前。

设计要点：

- 显示当前任务标题。
- 显示练习进度条。
- 显示建议追问和目标表达。
- 目标表达可以被“填入示例回答”按钮调用。

### 对话练习区

用途：完成文本输入和音频上传。

设计要点：

- 音频上传放在输入之前，标记为可选。
- 提供两个快捷按钮：填入当前任务问题、填入示例回答。
- 文本输入框高度固定，避免长文本撑乱页面。
- 主操作按钮使用高识别度红色。

### 右侧反馈区

用途：让用户在一次提交后同时看到 AI 回复、纠错、评分。

设计要点：

- AI 回复使用绿色成功卡片，表示系统已完成角色回应。
- 纠错反馈先显示状态，再显示详细建议。
- 评分卡必须使用深色背景和高对比文字，避免白底白字。

### 评分面板

用途：展示综合评分和五维评分。

设计要点：

- 综合分使用大字号。
- 维度分使用固定深色卡片。
- 每项评分都带解释，不只显示数字。
- 进度条使用蓝绿渐变，便于截图展示。

## 交互流程

推荐演示路径：

1. 选择 `面试 Interview` 场景。
2. 保持难度为 `中等`。
3. 点击 `填入示例回答`。
4. 点击 `提交练习`。
5. 查看 AI 回复、纠错反馈和评分。
6. 点击 `生成课后总结`。
7. 切换到 `报告导出`，下载 Markdown 或生成 HTML。
8. 切换到 `学习统计`，展示趋势和历史。

## Canva 复刻步骤

如果需要在 Canva 制作比赛展示图，可以按以下步骤复刻：

1. 新建设计，尺寸选择 `1440 x 1000`。
2. 设置背景色为 `#0b1220`。
3. 左侧创建宽 `288px` 的侧栏，颜色 `#111827`。
4. 顶部创建 Hero 卡片，位置约为 `x=336, y=48, w=1056, h=166`。
5. 主内容使用双栏：
   - 左栏宽约 `612px`，放练习任务、上传、输入和按钮。
   - 右栏宽约 `396px`，放 AI 回复、纠错、评分。
6. 评分卡统一使用 `#0f172a` 背景、`#f8fafc` 标题、`#cbd5e1` 说明。
7. 使用蓝绿进度条表达分数。
8. 导出 PNG，用于比赛项目说明或展示 PPT。

## 设计验收标准

- 页面首屏能看出这是英语口语陪练工具。
- 用户不用阅读 README 也能知道下一步该输入或点击什么。
- 练习后反馈区不混乱，AI 回复、纠错和评分互相分开。
- 评分文字在深色主题下清晰可读。
- 文档中能解释视觉设计、交互设计和信息架构。
- Figma 文件或 Canva 展示图能作为比赛项目说明中的设计成果证明。
