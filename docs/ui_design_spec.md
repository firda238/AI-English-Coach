# AI-English-Coach V1.0 UI 设计说明

## 设计目标

本轮 UI 优化围绕三个目标展开：

- 页面好看：建立统一的深色视觉系统、清晰的层级、稳定的卡片和高对比评分组件。
- 页面好用：增加快捷填入按钮、明确空状态、把练习任务和反馈结果放在用户视线内。
- 页面清楚：把系统拆成练习中心、历史记录、学习统计、报告导出、设置说明五个一级标签页。

## 视觉设计

页面采用深色学习工具风格，主色为深蓝黑，强调色为青蓝和绿色。深色背景用于承载学习场景，青蓝表示 AI 和练习引导，绿色表示进度和完成状态。评分卡使用固定深色背景、白色大号分数和蓝绿进度条，避免深色主题下默认 Streamlit 组件出现“白底白字”的可读性问题。

顶部 Hero 区展示产品名称、功能说明和四个关键状态：模式、场景、难度、进度。这样用户打开页面后能立即知道自己在哪个练习状态中。练习任务卡使用进度条、当前任务、建议追问和目标表达，帮助用户理解下一步应该练什么。

## 交互设计

练习中心中保留文本输入和音频上传，同时新增两个快捷按钮：

- 填入当前任务问题：把当前练习步骤的建议问题填入输入框，适合快速测试和比赛演示。
- 填入示例回答：把目标表达和示例句组合成回答，适合演示 AI 回复、纠错和评分流程。

空状态不再只显示一句提示，而是说明“下一步会出现什么”。例如对话历史为空时，提示用户先完成练习任务；评分为空时，提示提交后会出现综合分和五维评分。

## 信息架构

主界面拆为五个标签页：

1. 练习中心：当前任务、输入、对话、AI 回复、纠错、评分和本次总结。
2. 历史记录：筛选、查看、下载和删除历史 JSON。
3. 学习统计：总次数、平均分、最高分、场景分布、趋势和高频问题。
4. 报告导出：选择当前总结或历史总结，导出 Markdown / HTML。
5. 设置说明：API、语音转写和测试命令。

这种结构避免把所有功能堆在一个长页面中，更适合截图展示和比赛展示说明。

- 主卡片色：#101827 / #111f32。
- 边框色：#243244。
- 主文字：#f8fafc。
- 次级文字：#cbd5e1。
- 强调色：#38bdf8 和 #22c55e。
- 页面结构：左侧侧栏 + 顶部 Hero + 五个标签页 + 练习双栏布局。
