# AI-English-Coach 产品设计 QA 报告

## QA 目标

本报告用于检查当前 AI-English-Coach 页面是否满足本轮产品设计目标：

- 页面好看：视觉设计统一、层级清晰、适合截图展示。
- 页面好用：用户能顺利完成场景选择、输入、提交、反馈、总结和导出。
- 页面清楚：信息架构分区明确，不把所有功能堆在一个长页面里。

本报告的依据包括：

- Streamlit 页面源码：`app.py`
- UI 设计说明：`docs/ui_design_spec.md`
- Figma / Canva 交付说明：`docs/figma_canva_handoff.md`
- Figma 可编辑设计稿：<https://www.figma.com/design/p7RyLn8dlM23u6Ica8ehsY>

- 本地 Canva 风格展示 PNG：`outputs/canva_showcase.png`
- 自动化测试：`tests/test_core.py`、`scripts/smoke_test.py`

## 总体结论

当前版本已经达到比赛项目的产品展示要求。页面不是简单的默认 Streamlit 表单，而是有清晰的学习工具结构：左侧栏提供上下文，顶部 Hero 提供状态总览，主区域用标签页拆分功能，练习中心采用输入区和反馈区双栏布局。用户完成一次练习后，可以连续看到 AI 回复、纠错反馈、五维评分和课后总结入口。

## 视觉设计检查

### 已满足项

1. 颜色系统统一
   - 页面背景使用 `#0b1220`。
   - 主卡片使用 `#101827` / `#111f32`。
   - 强调色使用 `#38bdf8` 和 `#22c55e`。
   - 主操作按钮使用 `#ff4b4b`，与深色背景形成明显对比。

2. 层级清晰
   - 顶部 Hero 使用大标题和状态卡片，首屏识别度高。
   - 练习任务卡放在输入区之前，用户先知道任务，再开始输入。
   - AI 回复、纠错反馈、评分分别使用不同区块，不混在一起。

3. 评分可读性修复
   - 评分总分卡和维度卡都强制使用深色背景。
   - 分数使用白色文本，说明文字使用浅灰色。
   - 进度条使用蓝绿渐变，截图时容易识别。

4. 设计交付完整
   - Figma 文件中包含主界面、设计 handoff 和 Canva 展示海报三张画板。
   - `docs/canva_showcase.html` 可以作为比赛项目说明中的视觉展示页。
   - `outputs/canva_showcase.png` 可以直接放入比赛项目说明或展示 PPT。

### 仍可优化项

1. Streamlit 原生组件仍有部分默认样式，例如文件上传框和图表区域。
2. 当前没有自定义图标系统；第一版为了稳定，没有引入额外 icon 依赖。
3. Figma 文件是可编辑设计稿，但不是从网页像素级捕获生成，因此与运行页面存在少量视觉差异。

## 交互设计检查

### 主流程

当前主流程为：

```text
选择场景 -> 选择难度 -> 查看练习任务 -> 输入或上传音频 -> 提交练习 -> 查看 AI 回复 -> 查看纠错反馈 -> 查看五维评分 -> 生成课后总结 -> 导出报告
```

这个流程符合英语口语陪练工具的实际使用顺序，也适合比赛展示演示。

### 已满足项

1. 降低开始成本
   - `填入当前任务问题` 可以快速把任务问题填入输入框。
   - `填入示例回答` 可以快速生成一次完整演示。
   - 空状态会说明提交后将出现什么内容。

2. 失败状态可控
   - 空输入会提示，不会崩溃。
   - 无 API Key 自动进入本地演示模式。
   - 语音模型不可用时会降级到文本输入。

3. 输出反馈连续
   - 提交一次练习后，AI 回复、纠错、评分同屏出现。
   - 用户不需要跳转页面才能理解结果。

4. 导出路径完整
   - 课后总结可以生成 Markdown。
   - 报告导出页可以生成 HTML 报告。
   - 历史记录可下载 JSON。

### 仍可优化项

1. 提交按钮目前未显式显示 loading 状态，Streamlit 会有默认刷新反馈。
2. 音频上传后真实转写依赖本地 Whisper 模型，首次模型下载可能较慢。
3. 如果后续要做商业级产品，需要加入用户进度保存、账号和更细的练习任务状态。

## 信息架构检查

### 当前结构

```text
左侧栏
- 场景选择
- 难度选择
- 模式状态
- 练习目标
- 常见问题与推荐表达
- 最近保存记录

顶部区域
- 产品标题
- 项目说明
- 模式 / 场景 / 难度 / 进度

主标签页
- 练习中心
- 历史记录
- 学习统计
- 报告导出
- 设置说明
```

### 已满足项

1. 高频操作集中在练习中心。
2. 低频管理功能移到历史记录和报告导出。
3. 统计信息单独成页，不干扰练习流程。
4. 设置说明只放命令和模式说明，不占用主练习界面。
5. 左侧栏保留场景和目标，用户切换标签页时仍知道当前上下文。

### 仍可优化项

1. 历史记录较多时可以加入日期范围筛选。
2. 学习统计后续可以增加按场景对比的雷达图。
3. 设置说明可以扩展为“部署说明”和“常见问题”两个区域。

## 最终评价

当前产品设计已经达到“比赛展示级完整版本”。它不是复杂商业架构，但在 24 小时比赛项目语境下已经具备：

- 可运行的真实页面；
- 清晰的学习主流程；
- 统一的视觉风格；
- 比赛项目说明可解释的信息架构；
- Figma 可编辑设计稿；
- Canva 风格 HTML 展示页和 PNG 展示图；
- 自动化测试和验收文档。

当前已按“先不做 Canva 在线链接”的范围完成本地项目、Figma 设计稿、Product Design 文档和报告展示材料。后续如果继续优化，优先级应为：

1. 用真实浏览器补更多演示截图，覆盖练习、历史、统计和报告导出。
2. 增加 loading 状态和更多微交互。
3. 接入真实发音评分或更细的本地语音特征分析。
4. 后续如果恢复 Canva 工作，再生成在线 Canva 设计稿。
